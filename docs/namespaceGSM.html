◆ process()

void GSM::process ( void *  context)

Handle communication with the modem.

Parameters
[in]contextdriver operating context.
Warning
assumes context can be statically cast to context_t;

The process loop handles communication with the modem and transitions between device states. If there is not a command already awaiting a response then the next command in the buffer is sent. Responses are handled based on the GSM::State. Any pending packets that exceed their timeout value are discarded and the error counter incremented.
If the error counter exceeds MAX_ERRORS the modem is assumed to be MIA and the driver returns to State::none. The error counter is reset whenever a response is received.

If the signal is lost the connection is assumed to be broken and the driver returns to State::offline. The user must call authenticate() and connect() again to re-establish.

State::none - wait for the modem to respond to an ATI query and transition to State::init.

State::init - query the SIM card and transition to State::offline or State::locked.

State::locked - SIM is locked, call unlock() with the password to transition to State::offline.

State::offline - wait for a signal and transition to State::online.

State::online - registered on network, wait for authenticate() to transition to State::authenticating.

State::authenticating - handle authenticate() and transition to State::ready on success.

State::ready - ready to GPRS, wait for connect() to transition to State::handshaking.

State::handshaking - handle connect() and transition to State::open on success.

State::open - socket is established, handle write() and listen for incoming data
